Project 003 · Sanitized Enterprise Case Study

Employee Offboarding Automation

Microsoft Power Automate · Microsoft Entra ID · Exchange Online · Distribution List Cleanup

Privacy note: This case study documents a production automation I designed and implemented in a prior enterprise environment. Client names, tenant details, user information, and screenshots are intentionally excluded.

Executive Summary

This project automated a repetitive Microsoft 365 offboarding task: removing departing employees from Exchange Online distribution lists. A scheduled Power Automate workflow reviewed members of a designated Offboarding group, identified the distribution lists associated with each user, and removed those memberships through Microsoft Graph requests.

The solution reduced manual administration, improved consistency, and helped prevent former employees from continuing to receive internal distribution-list communications.

My Role

I independently designed, built, tested, troubleshot, and documented the workflow. My work included defining the offboarding control group, retrieving its members, querying distribution lists, handling nested iteration, configuring authenticated Microsoft Graph requests, resolving JSON and URI errors, and validating successful membership removal.

Business Challenge

Offboarding administrators often need to locate and remove a departing employee from multiple distribution lists. Completing this work manually is repetitive, time-consuming, and easy to overlook—especially when users belong to many lists across a large Microsoft 365 environment.

Business Requirements

  • Use a designated Offboarding group as the source of users requiring cleanup.
  • Run automatically on a recurring schedule.
  • Retrieve every user currently assigned to the Offboarding group.
  • Identify applicable Exchange Online distribution lists.
  • Remove each departing user from matching distribution-list memberships.
  • Use a repeatable process that reduces manual effort and human error.
  • Document the workflow for support, maintenance, and future enhancement.

Solution Architecture

Control Group

A Microsoft 365 group named Offboarding served as the operational queue for users requiring distribution-list cleanup.

Automation Engine

A scheduled Power Automate cloud flow retrieved group members and processed each user through the cleanup logic.

Microsoft Graph

Authenticated HTTP requests were used to remove distribution-list memberships programmatically.

Workflow Overview

Scheduled trigger → Find the Offboarding group → Retrieve group members → Process each user → Retrieve eligible distribution lists → Remove matching memberships through Microsoft Graph → Record a successful run.

Automation Process

1. Scheduled Execution

The cloud flow ran on a recurring schedule so administrators did not need to launch the cleanup process manually.

2. Offboarding Group Lookup

The workflow located the Microsoft 365 group by its display name and retrieved the group's unique identifier.

3. Member Retrieval

The flow retrieved all users currently placed in the Offboarding group and processed each member individually.

4. Distribution List Discovery

The workflow queried mail-enabled, non-security groups to identify the distribution lists that required membership evaluation.

5. Automated Membership Removal

For each applicable user and distribution list, the workflow issued an authenticated Microsoft Graph HTTP DELETE request to remove the user's membership.

Key Technical Challenges

Nested Iteration

Challenge: The automation needed to process multiple users and multiple distribution lists without mixing values between loops.

Resolution: Structured the workflow with clearly separated Apply to each actions and referenced the correct current-item values at each level.

JSON Response Handling

Challenge: Early versions failed when expected response properties were referenced incorrectly.

Resolution: Corrected the response handling and removed unnecessary parsing steps so the flow used the returned values reliably.

Microsoft Graph URI Construction

Challenge: Membership-removal requests failed when the Graph resource path was incomplete or malformed.

Resolution: Corrected the resource URI and dynamically inserted the required group and user identifiers.

Authentication and Permissions

Challenge: The HTTP action required the correct Microsoft Entra ID resource configuration and sufficient permissions.

Resolution: Configured the connection to authenticate against Microsoft Graph and validated the service account's access.

Business Impact

  • Reduced repetitive distribution-list cleanup work.
  • Improved consistency across employee offboarding.
  • Lowered the risk of overlooked memberships.
  • Created a repeatable, scheduled administrative process.
  • Improved security by removing unnecessary access to internal email communications.
  • Provided a foundation for broader lifecycle automation.

Project Outcome

The final workflow successfully processed users assigned to the Offboarding group and removed their distribution-list memberships through Microsoft Graph. The automation was stabilized after resolving nested-loop, response-handling, authentication, and URI-construction issues.
